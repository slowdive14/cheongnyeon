# Safety & Domain Audit — 안전·신뢰 감사

이 도구는 취약한 청년을 다룬다. 안전 위반은 정확성 버그보다 위험하다. **존재 확인이 아니라 동작 경로 추적**으로 감사한다 — 입력에서 출력까지 코드·테스트를 따라간다. 의심되면 보수적으로 High.

## 감사 체크리스트

### 1. 위기 라우팅 우선 (최고 위험)
- 위기 신호(명시적 + **의심 포함, 거짓양성 허용**) 입력 시 정책 깔때기보다 **안전자원이 항상 최우선·최상단** 라우팅되는가.
- 안전자원에 자살예방상담 109, 정신건강 위기상담 1577-0199가 포함되는가.
- `traverse`·`crisisGuard` 양쪽에서 위기가 정책 결과로 새는 경로가 없는가(자유입력 경로 포함).
- LLM이 치료적 조언 텍스트를 생성하지 않는가(안전자원 안내로만).

### 2. 자격 보수성
- 애매한 자격이 탈락이 아니라 `"확인 필요"`로 처리되는가.
- 필수조건 누락 시 보수 처리되는가(통과로 가정 금지).
- 경계값(나이 34/35, 소득 경계)이 의도대로 동작하는가.

### 3. LLM 그라운딩
- `explain`에 실제 정책 record가 주입되는가.
- 모델이 없는 정책·없는 필드를 날조하지 못하게 입력·후처리가 막는가.

### 4. 고지·투명성
- 모든 결과 카드에 **'추정 · 최종 자격은 시행기관 공고 기준'** 고지가 있는가.
- 원문 링크 + 데이터 최종 업데이트(신선도) 시각이 노출되는가.
- 전 결과에 위기 안내 푸터가 있는가.

### 5. 막힘 처리
- `blocked`가 청년 화면에 빨간 결과로 노출되지 않고 **대안 갈래**로 유도되는가(계산은 하되 숨김).

### 6. Graceful Degradation
- Gemini 키 없음/실패 시에도 버튼 흐름으로 **완전 동작**하는가.

## 판정 방법
- 각 항목을 코드·테스트로 교차 확인. 코드만으로 확신 불가 시 **"검증 불가"**로 표시하고 실행 테스트 요청 — 통과로 가정 금지.
- 위반은 **High/Med/Low**. 1·3번(위기·그라운딩)은 의심도 High.
- 누락된 안전 테스트는 어떤 시나리오가 비었는지 구체적으로 지목(예: "자유입력 위기어 → 안전 라우팅 테스트 없음").

## 출력
`_workspace/03_safety_phase{N}.md` — 위험도별 위반·미검증 + 필요한 추가 테스트. High 1개라도 있으면 머지 차단 권고.

## 근거
plan의 Success Criteria(위기 우선·고지·그라운딩)와 Risk Assessment(위기 오안내 High, 자격 오판정 High)가 SSOT. 상담사 간담회 근거는 프로젝트 메모리 `project-cheongnyeon-policy-site` 참조.
