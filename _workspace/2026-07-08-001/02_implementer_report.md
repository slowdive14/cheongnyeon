## H1 수정 — 전각 물결/`-` 구분자·last-match (code-reviewer blocker, 2026-07-08)

### 문제
`coarseOperationEnd`가 `text.indexOf('~')`(ASCII만)로 종료부를 분리 + 세그먼트 first-match를 써서
전각 물결·`-` 구분자 기간에서 **시작월을 종료로 오인** → 활성 사업이 3월 마감으로 기록돼 은폐.
- "2023. 3. ～ 2023. 12." → "2023-03-31" (기대 2023-12-31)
- "2023. 3. - 2023. 12." → "2023-03-31" (기대 2023-12-31)
- "2026. 5. ～"(전각, 끝 열림) → "2026-05-31" (기대 미도출 — 핵심 보수 불변식 위반)

### 수정 내용 (`src/data/seoulClient.ts`)
1. **물결 인식 확장**: `TILDE_CHARS = '~～∼〜'`(ASCII·전각 `～`·변형 `∼` U+223C·`〜` U+301C),
   `text.indexOf('~')` → `text.search(TILDE_SPLIT_RE)`. 전각 끝-열림도 종료부가 빈 세그먼트가 되어 미도출.
2. **first-match → last-match**: 신규 `lastPlausibleMatch(segment, source)` — 전역 매칭 중
   `isPlausibleYear` 통과하는 **마지막**(더 늦은) 것 반환. `coarseEndFromSegment`의 세 형식
   (연.월.일 / 연.월 / 연도만) 모두 이를 사용. `-` 구분자 기간은 분할 없이도 뒤쪽 연.월이 종료로 잡힘.
   빈 매칭 시 `lastIndex += 1`로 무한루프 방어.
3. 이전 TODO(무효-유효 연도 순서 오탐)도 last-match 전환으로 부수 개선 — 뒤쪽 유효 연도가 잡힘
   (단 여전히 앞선 무효 연도가 긴 숫자열이면 `isPlausibleYear`가 개별 배제).

### 추가 테스트 (5건, 전부 통과)

- `-` 구분자 기간: "2023. 3. - 2023. 12." → 2023-12-31 (last-match)
