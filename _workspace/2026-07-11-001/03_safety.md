## [High-1] ②`inheritedEndFromSegment`의 단일-숫자 폴백(mOnly)이 잔여 숫자를 월로 오독 → 상시 사업 오마감

### 동작 경로 (확인)
`deriveSeoulRecruit` → (apply 날짜 없음·상시) → `coarseOperationEnd` → 종료부 직접 도출 실패 → `inheritedEndFromSegment`. 이때 종료부의 "월.일"(md) 실패 시 `mOnly = /(\d{1,2})/.exec(endPart)`가 **종료부 아무 곳의 1~2자리 숫자**를 월로 채택한다. 운영기간 종료부가 실제로는 열려 있고(진행 중) 정원·기간·회차 등 잔여 숫자만 있는 경우, 그 숫자가 1~12이면 가짜 마감월이 만들어진다.

가짜 종료일 → `recruitStatus`가 `'closed'` → `eligibility.recruitAxis`가 `blocked('RECRUIT_CLOSED')` → 청년 화면 **완전 미노출**.

### 실행 프로브 결과 (deriveSeoulRecruit 직접 구동)
| 운영기간(상시 신청) | 출력 | 판정 |
|---|---|---|
| `2026. 2. ~ 5명 모집` | `endText 2026-05-31` | 오마감 — "5명"의 5를 5월로. **오늘(7/11) 기준 이미 과거 → blocked → 지금 은폐** |
| `2026. 3. ~ 6개월` | `endText 2026-06-30` | 오마감 — "6개월"의 6을 6월로(실제 ~9월). **과거 → 지금 은폐** |
| `2026. 5. ~ 소진 시(최대 100명)` | `endText 2026-10-31` | 오마감 — "100"의 10을 10월로. 10월 이후 은폐 |
| `2026. 8. ~ 2차 모집` | `endText 2027-02-28` | 오탐(2차→2월+해넘김) — 이 건은 늦게 잡혀 은폐는 아니나 없는 마감 날조 |

대조로 정상 유지된 것: `2026. 5. ~`→상시, `2026. 3. ~ 예산 소진시`→상시, `2026. 3. ~ 30일 이내`→상시(30>12 배제), `2024. 1. ~ 12.`→2024-12-31(의도한 버그픽스), `2026. 11. ~ 3.`→2027-03-31(해넘김 정상).

### 왜 High
- 이 변경이 **새로 도입한** 은폐 경로다(변경 전엔 위 상시건 전부 노출). 과업이 지목한 핵심 리스크(오은폐)와 정확히 일치.
- 종점이 `blocked`라 "확인 필요"도 아닌 **완전 소멸** — 취약 청년이 자격 있는 마음건강/지원 정책을 놓친다.
- 프로브 2건은 오늘 기준 이미 과거 → 해당 데이터가 있으면 현재 은폐.

### 검증 갭 (정직 고지)
`data/cache/policies.json`에 seoul 소스 레코드 0건(전량 ontong)이라 **실 서울 운영기간 문자열 표본으로 빈도 확인 불가**. 코드 동작은 실행으로 확정, 실데이터 발생률은 미확인. 보수 원칙("의심되면 High", "통과로 가정 금지")에 따라 High 유지. 라이브 서울 fetch 표본으로 종료부 패턴 재검 권고.

### 권고 수정(좁게)
`inheritedEndFromSegment`의 mOnly 폴백을 **날짜 형태 숫자만** 수용하도록 제한:
- 채택 조건: 숫자가 날짜 구분자(`.`/`-`/`/`)·`월`·문자열 끝에 인접(예 "12." "12월"). 즉 의도한 버그픽스 "…~ 12."는 통과.
- 배제: 숫자 직후가 계량 명사(명/개월/년/차/회/원/만 등)면 월로 쓰지 않음("5명" "6개월" "2차" 배제).
- 배제 시 null 반환 → `isAlways`면 상시 유지(오은폐 없음). 기존 그린 테스트(2024. 1. ~ 12. 등) 전부 보존.
- 회귀 테스트 추가: `('상시','2026. 2. ~ 5명 모집')→{text:'상시'}`, `('상시','2026. 3. ~ 6개월')→{text:'상시'}`, `('상시','2026. 5. ~ 소진 시(최대 100명)')→{text:'상시'}`.

## ② 나머지 감사 포인트 — 통과
- **(a) 진행 중 상시 노출**: `2026. 1. ~ 12.`→2026-12-31(미래→now 노출), `2026. 5. ~`(끝 열림)→상시, 운영기간 없음+상시→상시. 세 케이스 모두 노출 유지 확인.
- **(b) 연도 상속·해넘김 +1 방향**: 로직상 연도 감산 경로 없음. 해넘김은 +1(늦은 마감)만 → 보수 방향 정확. 단 위 High-1은 "월 오독"이라 별개(연도 방향 아님).
- **(c) 신청기간 full-date 우선 회귀**: `deriveSeoulRecruit` 첫 분기(`seoulRecruitDates(applyText)`)가 최우선 즉시 반환으로 유지. 거친/상속 파서는 operationText에만 호출 — 신청기간 오염 없음. 회귀 없음.

## ④ dedupeYearVariants — 통과
- **(a) 서로 다른 사업 오병합**: 실데이터 470건 정규화 키 충돌 9건 전수 확인 — 전부 **같은 사업**의 연도/공백/`서울시→서울` 변형(예: `서울 청년수당`↔`서울시 청년수당`, `2026 매장유산…`↔`2025 매장유산…`). 서로 다른 사업 병합 0건. `normalizeName`은 괄호 문자만 제거하고 지역 단어(성북구/중랑구)는 한글로 보존 → 지역 분리 유지 확인. ORG_SYNONYMS는 서울특별시/서울시→서울 2건뿐(비공격적). 키 null(빈 정규화)은 그룹 제외 → 깨진 제목 오병합 방지.
- **(b) 대표 역전(구판이 신판 은폐)**: `pickRepresentative`가 상태 우선(now>soon>review) → 동일 상태서 연도 큰 것 → updatedAt. 살아있는(now) 판을 우선 노출, 동일 상태 내에선 신판 우선. 알려진 live 정책을 숨기는 역전 경로 없음. blocked는 dedupe 입력에서 제외되어 애초 경쟁 안 함.
- **(c) blocked 미전달·위기/고지 무접촉**: `ResultList`가 now/soon/review만 `dedupeYearVariants`에 전달, blocked 미전달 확인. 유틸은 filter만 수행 — 라벨·상태·고지·위기 표면 무변경. 전멸 금지(그룹당 대표 1개 항상 잔존) 유지.
- 부수 변경 `displayCategory`(카테고리 중복 토큰 제거)는 표기만 정리, 색 로직(categoryTag)은 원문 사용 — 안전 무관.
