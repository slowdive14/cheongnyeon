# DESIGN.md — 청년정책 진단 디자인 SSOT

> 이 문서는 UI 작업의 단일 진실 원천이다. 모든 UI 구현·수정(Phase D·E·F 포함)은 이 문서를 근거로 한다.
> 문서와 코드가 어긋나면 **이 문서를 먼저 고치고** 코드를 따라가게 한다.
> 확정 근거: 2026-07-02 사용자 승인 시안 2종(메인 화면·펼침 카드) + PLAN_ops-and-seoul-expansion.md Phase E·F.

## 1. 원칙

1. **다정함이지 발랄함이 아니다.** 핵심 사용자는 지치고 무기력한 상태로 들어온다. 통통 튀는 모션·과한 이모지·명랑한 감탄은 부담이다. 차분한 온기.
2. **안전 표면은 톤 전환 금지.** 위기 배너(SafetyBanner)·위기 푸터(CrisisFooter)·"추정" 고지(DisclaimerNote)·원문 링크는 진지한 문구·형태를 유지한다. 여기가 흔들리면 신뢰가 무너진다.
3. **개수와 사실로 신뢰를 만든다.** "N개를 찾았어요", "어제 업데이트된 전국 N개 정책" — 장식이 아니라 실데이터를 노출하는 것이 신뢰 디자인이다.
4. **관공서 서식이 아니라 대화.** 시스템이 주어인 문장 금지("검색 결과가 없습니다") → 사람이 말 거는 문장("이 방향으론 못 찾았어요. 이런 쪽은 어때요?").
5. **접근성은 KRDS 바닥선.** 공공 데이터 서비스로서 [KRDS](https://www.krds.go.kr/) 접근성 기준(대비·포커스·라벨)을 바닥선으로 참고한다. 단 시각 아이덴티티는 KRDS를 따르지 않는다(관공서 톤 회귀 금지).

## 3. 모양·타이포

> **비주얼 리프레시(2026-07-08, 사용자 시안 반영)**: 아래 "플랫" 원칙은 완화됐다. 사용자 승인 시안(`청년정책 요즘어때요 개선.dc.html`)의 **따뜻한 라디얼 그라데이션 배경·부드러운 그림자 카드·영역별 파스텔 그라데이션 아이콘 칩·클레이 그라데이션 CTA**를 채택한다. 색/간격은 시안 값 유지. 단 안전 표면(위기·추정·원문)의 진지함과 원칙 1(과한 모션 금지)은 불변.

- **라운드**: 카드 18~22px, 입력면 22px, 버튼·알약·배지 pill(999px). 아이콘 스퀘어 13px.
- **보더/면**: 헤어라인(#F0E6D8 등 웜) + 부드러운 그림자 허용(예: `0 18px 40px -18px rgba(160,90,50,.35)`). 배경은 라디얼 그라데이션 캔버스.
- **폰트**: Pretendard(웹폰트) 권장, 폴백 시스템 스택. 헤드라인은 800(extrabold) 허용.
- **크기**: 인사 30px·800, 결과 헤드라인 26px·800, 카드 제목 17.5px·800, 본문 14~15px, 보조 12~13px.
- **간격**: 섹션 간 18~26px, 카드 내부 패딩 17~18px. 여백이 다정함의 절반이다.
- **모션**: 페이드·floatIn(미세 상승) 150~300ms. 바운스·스프링 금지(원칙 1).
- **강조 그라데이션**: 인사·결과 숫자 텍스트 `linear-gradient(120deg,#E0733F,#C63C7A)` 클립. CTA `linear-gradient(135deg,#D2703F,#B84A2C)`. 컨텍스트/맞춤 배지 초록 `#D8F0C6→#B9E9D4`.
- **액션 언어 통일(2026-07-10)**: 주 행동 버튼(내 정책 찾기·카드 "신청 페이지 열기")은 전부 클레이 그라데이션 CTA + 웜 그림자. 다크(#2B2622) 채움 버튼 금지(투박 — 사용자 피드백). §2의 "클레이 채움 한 화면 1개"는 페이지 레벨 기준이며 카드 반복 CTA(카드당 1개)에는 적용하지 않는다.

### 3.1 반응형 레이아웃 (2026-07-10 확정)

> 모바일 우선. 데스크톱은 결과 화면에서만 폭을 넓혀 2열 그리드로 정보 밀도를 올린다. 홈은 모바일 시안 그대로가 데스크톱에서도 안정적이므로 중앙 정렬 유지.

- **외곽 셸 기준선 통일(2026-07-11 확정)**: 홈·결과 **비위기 화면의 `<main>` 외곽 셸은 동일 폭 클래스**(`max-w-[420px] lg:max-w-5xl`)를 공유한다. 외곽 프레임이 화면 전환에도 리사이즈되지 않아 배경·좌우 모서리가 튀지 않는다(2026-07-11 사용자 피드백: "검색창이 결과 뜨고 갑자기 확 넓어져 이상"). 시각 폭 차이는 **내부 콘텐츠 래퍼**로 만든다.
- **홈 화면**: 외곽 셸은 위 기준선. 내부 콘텐츠는 데스크톱에서 `lg:max-w-[420px]` 중앙 고정(모바일은 외곽 폭 그대로) → 시각 결과는 기존 홈과 동일(420px 중앙). 히어로 자유입력 카드 유지.
- **결과 화면(노출 카드 ≥1)**: 외곽 셸은 위 기준선. 내부 콘텐츠는 데스크톱 전폭(5xl) 사용, 결과 카드 목록을 `lg:grid-cols-2` 그리드로 렌더. 모바일·태블릿은 1열(현행). 카드 높이 불균등은 grid 자연 흐름 허용(masonry 불필요, `lg:items-start`로 행 늘어남 방지). 자유입력은 **compact 재검색 바**(§4)로 축약 — 거대한 히어로 입력이 결과 화면까지 늘어나지 않게 한다.
- **상단부(브랜드 바·헤드라인·프로필 알약·자유입력)와 하단부(동행·위기 푸터·신청함)**: 그리드 밖이되 **그리드와 같은 폭**(컨테이너 전폭). 구획별 너비가 제각각이면 좌우 모서리가 어긋나 어수선하다 — 중앙 좁힘 레일 금지(2026-07-10 사용자 피드백으로 `lg:max-w-xl` 레일 제거).
- **위기 화면**: 불변(`max-w-xl` 단일 컬럼, SafetyBanner 단독). 반응형 대상 아님(원칙 2·§7).

### 3.2 카드 푸터 줄바꿈 (2026-07-10 확정)

- 결과 카드 하단 메타 줄(최종 업데이트 · 저장 버튼 · 신청 페이지 열기 CTA)은 `flex-wrap`으로 좁은 폭에서 자연 줄바꿈: 날짜/저장 한 줄 + CTA 전폭 한 줄. 넓은 폭(`sm` 이상)에서는 한 줄 유지.
- 저장 버튼·날짜 텍스트는 `whitespace-nowrap`로 글자 세로 꺾임 방지. 터치 타깃 44px 및 기존 카피는 불변(레이아웃만).

| 펼침 헤더(F-⑤) | (없음) | **제출 서류 (원문 그대로)** — 정책 원문 documentsText 발췌만 노출(가공·요약·날조 금지). **표시 정리(줄 분리)는 허용, 글자 불변**: `formatDocumentsText`가 앞머리 기호 노이즈 제거와 줄 나누기만 수행(문자 추가·수정·재배열·요약 금지 — 속성 테스트로 기계적 잠금). 세그먼트 1개면 현행 `whitespace-pre-wrap` 문단 하나로 원문 줄바꿈 보존 |
| 서류 사전 매칭(F-⑤) | (없음) | 발췌문에 이름이 **등장하는 서류만** 발급처 안내(예: "주민등록등본 — 정부24 · 무료 · 약 5분"). 미등장 서류는 미노출(필요 서류 암시·단정 금지). 수수료 0→무료·null→확인 필요, 소요 null→미표기 |
| 원문 확인 프레임(F-⑤) | (없음) | **신청 전 반드시 원문에서 확인해요** — 제출 서류는 정책마다 달라 원문이 최종 근거(정책별 서류 단정 금지) |

**금지어·금지 패턴**: 시스템 주어("~할 수 없습니다"), 느낌표 남발, "간단히/쉽게"(무기력 사용자에게 압박), 자격 단정("자격이 됩니다/안 됩니다" — 안전 불변). 응원 카피는 절제(체크 1회당 1줄 이내, 과잉 금지).

## 6. 접근성 바닥선 (KRDS 참조)

- 텍스트 대비 4.5:1 이상(뮤트 텍스트도 큰 글자 기준 3:1 이상), 색만으로 상태 구분 금지(배지에 텍스트 라벨 필수).
- 모든 입력에 `<label>` 연결(기존 관용 유지), 포커스 링 가시(2px, clay 또는 기존 sky→clay 전환).
- 터치 타깃 최소 44×44px(칩·버튼·알약).
- 키보드만으로 전 흐름 조작 가능(Enter 전송·Shift+Enter 줄바꿈 유지).

## 7. 안전 불변 조항 (UI 작업 시 절대 규칙)

1. 위기 상태 → SafetyBanner 단독 렌더(입력·결과·예시·프로필·동행 UI 전부 미렌더). 어떤 리디자인도 이 불변식을 못 건드린다.
2. "추정" 고지·원문 링크·신선도는 모든 결과 카드에 유지.
3. 상태 라벨 문구 변경 시 **기존 테스트의 라벨 단언 전면 갱신 동반**(라벨은 test/integration/funnel/* 에서 단언됨).
4. 자격 단정으로 읽히는 표현 금지 — 체크리스트 ✓는 "충족/제한 없음" 사실 서술로만(자격 단정 금지). '추정' 성격은 DisclaimerNote가 1회 고지(항목별 "(추정)" 중복 표기 안 함). 소득 pass는 무관 정책이므로 "소득 제한 없음".
5. UI 변경은 safety-domain-auditor 감사 대상(안전 표면 접촉 시).
